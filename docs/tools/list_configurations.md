# list_configurations

List EDT launch configurations (runtime client + Attach + other 1C types) with their running state. This is the discovery step before debug_launch / run_yaxunit_tests / debug_yaxunit_tests / update_database: use the returned 'name' as their launchConfigurationName. Use type='attach' for server-side debug setups, type='client' for 1C:Enterprise client configs, or type='all' (default). Full parameters and examples: call get_tool_guide('list_configurations').

## Parameters
| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| type | ‚Äî | string (one of: attach, client, all) | Filter by config kind; default 'all'. e.g. 'attach' for server-side debug. |
| projectName | ‚Äî | string | Optional exact project-name filter. |

## Guide
Lists EDT launch configurations ‚Äî runtime client, Attach (RemoteRuntime / LocalRuntime), and any other config in the 1C/EDT namespace ‚Äî together with their current running state. Once the client knows the exact `name`, it can target that configuration by name without juggling applicationId/project pairs.

## When to use
- Discovery step before `debug_launch`, `run_yaxunit_tests`, `debug_yaxunit_tests` and `update_database` ‚Äî copy the returned `name` into their `launchConfigurationName`.
- See whether a configuration is already running (and whether it is paused on a breakpoint) before launching a second client.

## Server-side debug workflow
1. `list_configurations({type: "attach"})` ‚Äî see available Attach configs, their infobase aliases, and whether any is already running.
2. `debug_launch({launchConfigurationName: ...})` ‚Äî attach to it.
3. `set_breakpoint` ‚Üí `wait_for_break` ‚Üí standard debug flow.

## Parameter details
- `type` ‚Äî filter by config kind:
  - `attach`: RemoteRuntime + LocalRuntime (server-side debug: HTTP services, background jobs).
  - `client`: RuntimeClient (1C:Enterprise client configs). The aliases `runtime` and `runtimeClient` are also accepted.
  - `all` (default): any 1C/EDT launch config. An unknown value is treated permissively as `all`.
- `projectName` ‚Äî optional; keeps only configs whose project attribute equals this value exactly. Omit to list across all projects.

## Result fields (per entry)
- `name` ‚Äî the configuration name; this is the value to pass as `launchConfigurationName` downstream.
- `type` ‚Äî the launch-config type id.
- `attach` ‚Äî boolean, true for Attach configs.
- `applicationId` ‚Äî real applicationId, or a synthetic `attach:<name>` for Attach configs (present only when known).
- `project`, `infobaseAlias`, `debugServerUrl` ‚Äî present only when the config defines them.
- `running` ‚Äî boolean; when true, `mode` (debug/run) is added, and `suspended` is true when a thread is paused on a breakpoint.

## Notes
- Returns JSON: a `configurations` array plus a `count`.
- Only the `launchConfigurationName` mode of `debug_launch` can start an Attach session ‚Äî the `projectName + applicationId` mode reaches runtime-client configs only.
