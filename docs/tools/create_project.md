# create_project

Create a NEW 1C project in the EDT workspace. projectKind selects the kind: 'configuration' (standalone), 'extension' (bound to a base configuration), or 'externalObjects' (external data processors/reports). The name must not already exist as a project. standardChecks/commonChecks are applied only when com.e1c.v8codestyle is installed. Full parameters and examples: call get_tool_guide('create_project').

## Parameters
| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| projectKind | yes | string | Kind of project to create: `configuration`, `extension`, or `externalObjects`. |
| name | yes | string | Name of the new Configuration object (or project name for externalObjects). Must be a valid 1C identifier. |
| projectName | — | string | EDT workspace project name. Default: extension → `<baseProjectName>.<name>`; configuration/externalObjects → `name`. |
| version | — | string | Platform version string, e.g. `8.3.27` (configuration and externalObjects only). Default: Version.LATEST. |
| baseProjectName | — | string | Name of the BASE configuration project (required for extension; rejected for other kinds). |
| prefix | — | string | NamePrefix for the extension (extension only). Default: empty string. |
| synonym | — | string | Human-readable synonym for the Configuration (configuration and extension only). |
| comment | — | string | Optional free-text comment (configuration and extension only). |
| purpose | — | string | Extension purpose: Customization (default), AddOn, or Patch (extension only). |
| compatibilityMode | — | string | Optional CompatibilityMode enum literal (e.g. `Version8_3_10`); extension only. |
| scriptVariant | — | string | Script variant: Russian (default) or English. configuration: sets Configuration+Language. externalObjects: applied post-create. extension: rejected. |
| standardChecks | — | boolean | Enable 1C:Standards BSL checks (default true). Applied only when com.e1c.v8codestyle is installed. |
| commonChecks | — | boolean | Enable common project-level BSL checks (default true). Applied only when com.e1c.v8codestyle is installed. |
| autoSortTopObjects | — | boolean | Reserved for future use; accepted but not yet applied in this release. |

## Guide
Creates a new 1C project of the selected kind in the EDT workspace.

## When to use

- **configuration** — standalone 1C configuration project for metadata authoring.
- **extension** — configuration extension to override metadata objects or add BSL interceptors.
- **externalObjects** — external data processors/reports project.

## Workflow (extension)

1. `create_project` (kind=extension) — create the extension and bind it to the base.
2. `adopt_metadata_object` — adopt the objects/members you want to override.
3. `create_metadata` / `modify_metadata` on the adopted objects — extend the behavior.
4. `delete_project` — remove the extension project when it is no longer needed.

## Gotchas

- **Project name must be new.** Rejected if a workspace project with the computed name already exists.
- **Extension: base must be a configuration, not an extension.**
- **After creation, wait for `state=ready`** before calling `adopt_metadata_object`.
- **autoSortTopObjects is not yet applied.** See `codestyle.autoSortNote` in the response.
